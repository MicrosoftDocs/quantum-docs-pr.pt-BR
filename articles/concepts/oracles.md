# <a name="quantum-oracles"></a><span data-ttu-id="8506a-180">Oracle Quantum</span><span class="sxs-lookup"><span data-stu-id="8506a-180">Quantum Oracles</span></span>

<span data-ttu-id="8506a-181">Um Oracle $ O $ é uma operação de "caixa preta" que é usada como entrada para outro algoritmo.</span><span class="sxs-lookup"><span data-stu-id="8506a-181">An oracle $O$ is a "black box" operation that is used as input to another algorithm.</span></span>
<span data-ttu-id="8506a-182">Muitas vezes, essas operações são definidas usando uma função clássica $ f: \\ { 0, 1 \\ } ^ n \to \\ { 0, 1 \\ } ^ m $ que usa $ uma $ entrada binária de n bits e produz $ uma $ saída binária de bit m.</span><span class="sxs-lookup"><span data-stu-id="8506a-182">Often, such operations are defined using a classical function $f : \\{0, 1\\}^n \to \\{0, 1\\}^m$ which takes an $n$-bit binary input and produces an $m$-bit binary output.</span></span>
<span data-ttu-id="8506a-183">Para fazer isso, considere uma entrada binária específica $ x = (x_ { 0 } , x_ { 1 } , \dots, x_ { n-1 } ) $ .</span><span class="sxs-lookup"><span data-stu-id="8506a-183">To do so, consider a particular binary input $x = (x_{0}, x_{1}, \dots, x_{n-1})$.</span></span>
<span data-ttu-id="8506a-184">Podemos rotular Estados qubit como $ \ket { \vec { x } } = \ket { x_ { 0 } } \otimes \ket { x_ { 1 } } \otimes \cdots \otimes \ket { x_ { n-1 } } $ .</span><span class="sxs-lookup"><span data-stu-id="8506a-184">We can label qubit states as $\ket{\vec{x}} = \ket{x_{0}} \otimes \ket{x_{1}} \otimes \cdots \otimes \ket{x_{n-1}}$.</span></span>

<span data-ttu-id="8506a-185">Nós podemos primeiro tentar definir $ o $ para que $ o \ket { x } = \ket { f (x) } $ , mas isso tenha alguns problemas.</span><span class="sxs-lookup"><span data-stu-id="8506a-185">We may first attempt to define $O$ so that $O\ket{x} = \ket{f(x)}$, but this has a couple problems.</span></span>
<span data-ttu-id="8506a-186">Em primeiro lugar, $ f $ pode ter um tamanho diferente de entrada e saída ( $ n \ne m $ ), de modo que aplicar $ $ o seria alterar o número de qubits no registro.</span><span class="sxs-lookup"><span data-stu-id="8506a-186">First, $f$ may have a different size of input and output ($n \ne m$), such that applying $O$ would change the number of qubits in the register.</span></span>
<span data-ttu-id="8506a-187">Em segundo lugar, mesmo se $ n = m $ , a função pode não ser invertível: se $ f (x) = f (y) $ para alguns $ x \ne y $ , em seguida, $ o \ket { x } = o \ket { y, } $ mas $ o ^ \dagger o \ket { x } \ne o ^ \dagger o \ket { y } $ .</span><span class="sxs-lookup"><span data-stu-id="8506a-187">Second, even if $n = m$, the function may not be invertible: if $f(x) = f(y)$ for some $x \ne y$, then $O\ket{x} = O\ket{y}$ but $O^\dagger O\ket{x} \ne O^\dagger O\ket{y}$.</span></span>
